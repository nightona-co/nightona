## Configuration in code

To configure your environment in code, use the `NightonaConfig` class. The `NightonaConfig` class accepts the following parameters:

- `api_key`: Your Nightona [API Key](/docs/api-keys)
- `api_url`: URL of your [Nightona API](/docs/en/tools/api)
- `target`: Target region to create the Sandboxes on (`us` / `eu`)

<Tabs syncKey="language">
<TabItem label="Python" icon="seti:python">

```python
from nightona import NightonaConfig

config = NightonaConfig(
    api_key="YOUR_API_KEY",
    api_url="YOUR_API_URL",
    target="us"
)
```

</TabItem>

## Environment variables

Nightona supports environment variables for configuration. The SDK automatically looks for these environment variables:

| Variable              | Description                                | Required |
| --------------------- | ------------------------------------------ | -------- |
| **`NIGHTONA_API_KEY`** | Your Nightona API key.                      | Yes      |
| **`NIGHTONA_API_URL`** | URL of your Nightona API.                   | No       |
| **`NIGHTONA_TARGET`**  | Nightona Target to create the sandboxes on. | No       |

### Shell

Set environment variables in your shell using the following methods:

<Tabs syncKey="shell">
<TabItem label="Bash/Zsh" icon="seti:shell">

```bash
export NIGHTONA_API_KEY=your-api-key
export NIGHTONA_API_URL=https://your-api-url
export NIGHTONA_TARGET=us
```

</TabItem>
<TabItem label="Windows PowerShell" icon="seti:powershell">

```bash
$env:NIGHTONA_API_KEY="your-api-key"
$env:NIGHTONA_API_URL="https://your-api-url"
$env:NIGHTONA_TARGET="us"
```

</TabItem>
</Tabs>

### .env file

Set the environment variables in a `.env` file using the following format:

```bash
NIGHTONA_API_KEY=YOUR_API_KEY
NIGHTONA_API_URL=https://your_api_url
NIGHTONA_TARGET=us
```

### Async Python SDK

Nightona provides an additional `NIGHTONA_HAPPY_EYEBALLS_DELAY` environment variable for HTTP transport tuning in the async Python SDK. Use it to reduce intermittent async connection failures, such as `aiohttp.ClientConnectorError`, that can occur on dual-stack (IPv4/IPv6) networks.

| Variable                           | Description                                                                 | Required |
| ---------------------------------- | --------------------------------------------------------------------------- | -------- |
| **`NIGHTONA_HAPPY_EYEBALLS_DELAY`** | Controls **`aiohttp`** Happy Eyeballs (IPv4/IPv6 connection race) delay in seconds. | No       |

- unset or empty: use `aiohttp` default behavior
- `none` (case-insensitive): disable the IPv4/IPv6 race
- non-negative float (for example `0.25`): set an explicit delay in seconds

```bash
# Disable Happy Eyeballs
export NIGHTONA_HAPPY_EYEBALLS_DELAY=none

# Or set an explicit delay in seconds
export NIGHTONA_HAPPY_EYEBALLS_DELAY=0.25
```

## Default values

If no configuration is provided, Nightona will use its built-in default values:

| **Option** | **Value**                           |
| ---------- | ----------------------------------- |
| API URL    | https://app.daytona.io/api          |
| Target     | Default region for the organization |
